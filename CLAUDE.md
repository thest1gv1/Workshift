# Workshift — контекст проекта

## Описание

Мобильное PWA приложение для сотрудников банка. Используется в течение рабочей смены для учёта клиентов и формирования отчёта в конце дня.

## Текущий стек

- Next.js 16 + TypeScript
- Tailwind CSS v4 + CSS Modules
- shadcn/ui
- Nano Stores
- свой vps (деплой)
- PostgreSQL
- pg + TypeScript (без ORM)
- Next.js API routes

## Будущий стек (этап 6+)

- Переход с чистого pg на Drizzle ORM (типобезопасные запросы, автомиграции)

## Экраны

- `/login` — авторизация
- `/` — текущая смена (главная)
- `/add` — добавить/редактировать клиента
- `/history` — список прошлых смен
- `/history/[id]` — смена с возможностью редактировать суммы по услугам
- `/stats` — статистика по месяцам
- `/settings` — настройки

## Флоу

1. Мастер авторизуется (один раз, потом автоматически)
2. В течение дня добавляет клиентов в текущую смену
3. В конце дня завершает смену — видит итоги, копирует отчёт
4. Смена уходит в историю, можно редактировать только суммы по услугам

## Режим обучения разработчика

1. не писать код за пользователя пытаться донести информацию так чтобы он понял и постарался сам, если уж он требует ответ и не может сам, помогай с кодом.

2. Старайся дать по возможности абрастрактный пример чтобы он больше понял

3. дополняй теорией

## Деплой

**Стек на VPS:**

- Ubuntu, Nginx как обратный прокси
- PM2 для управления процессом Node.js
- PostgreSQL локально на сервере

**Процесс обновления:**

1. `git push origin main` — пушишь код на GitHub
2. На VPS: `git pull` — тянешь изменения
3. `npm run build` — собираешь Next.js приложение
4. `pm2 restart workshift` — перезапускаешь процесс

**Миграции БД:**

- Файлы в папке `migrations/` с нумерацией `001_`, `002_` и т.д.
- Применяются вручную через: `sudo -u postgres psql -d workshift -c "SQL"`
- Всегда используй `IF NOT EXISTS` чтобы можно было запускать повторно
